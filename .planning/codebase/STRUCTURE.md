## Directory Purposes

**`convex/`:**
- Purpose: All backend logic — Convex schema, queries, mutations, actions
- Contains: `schema.ts` (currently the only hand-written file); future `queries.ts`, `mutations.ts`, `actions.ts` modules
- Key files: `convex/schema.ts` — the authoritative data model
- Note: Functions here are deployed to Convex cloud (or local dev backend), not to Vercel

**`convex/_generated/`:**
- Purpose: Auto-generated TypeScript types and JS stubs produced by `convex dev` / `convex deploy`
- Generated: Yes — regenerated every time schema or function signatures change
- Committed: Yes (required for type-checking before deploy)
- DO NOT edit manually

**`src/app/`:**
- Purpose: Next.js App Router — routes, layouts, pages
- Contains: `layout.tsx` (root), `page.tsx` (landing `/`), `globals.css`
- Future route segments will be added as subdirectories (e.g., `src/app/[billId]/`, `src/app/new/`, `src/app/dashboard/[billId]/`)

**`src/components/`:**
- Purpose: Shared React components used across multiple routes
- Contains: Currently only `ConvexClientProvider.tsx`
- Future: Bill builder form, chit display, claim rows, payment status, dashboard widgets

**`public/`:**
- Purpose: Static files served at the root URL
- Contains: Default Next.js SVG assets; will hold favicons, OG images

**`.convex/`:**
- Purpose: Convex local dev runtime data (SQLite DB, module blobs)
- Generated: Yes — created by `convex dev`
- Committed: No (`.convex/local/` is gitignored, `.convex/.gitignore` is committed)

## Key Entry Points

**App entry:**
- `src/app/layout.tsx` — root layout, wraps entire app in `ConvexClientProvider`

**Landing page:**
- `src/app/page.tsx` — route `/`

**Convex backend entry:**
- `convex/schema.ts` — data model definition
- `convex/_generated/api.d.ts` — typed API surface (consumed by client hooks)

**Convex client bootstrap:**
- `src/components/ConvexClientProvider.tsx` — creates `ConvexReactClient` using `NEXT_PUBLIC_CONVEX_URL`, provides Convex context

## Route Structure (App Router)

Current routes:
```
/                → src/app/page.tsx        (landing page)
```

Planned routes (per PRD §4 user flows):
```
/new             → src/app/new/page.tsx    (bill builder — organizer)
/bill/[billId]   → src/app/bill/[billId]/page.tsx   (member claim screen)
/dashboard/[billId] → src/app/dashboard/[billId]/page.tsx  (organizer dashboard)
```

All route pages that use Convex hooks or `localStorage` must be Client Components (`"use client"`). Static metadata can be declared in Server Components (layout files).

## How `convex/` Maps to `src/`

```
convex/schema.ts         defines tables
         ↓
convex/_generated/api.d.ts   exports typed `api` and `internal` references
         ↓
src/components/*.tsx     import from "convex/_generated/api"
                         call useQuery(api.queries.fn) / useMutation(api.mutations.fn)
```

The Convex client (`ConvexReactClient`) is instantiated once in `src/components/ConvexClientProvider.tsx` and provided via React context. Individual components import function references from `convex/_generated/api` and pass them to Convex hooks — no direct HTTP calls.

## Naming Conventions

**Files:**
- React components: PascalCase — `ConvexClientProvider.tsx`, `BillBuilder.tsx`
- Next.js special files: lowercase — `layout.tsx`, `page.tsx`, `globals.css`
- Convex backend files: camelCase or descriptive lowercase — `schema.ts`, `queries.ts`, `mutations.ts`

**Directories:**
- App Router route segments: lowercase, hyphenated — `bill/[billId]/`, `new/`
- Component folders: lowercase plural — `components/`

**CSS classes:**
- Tailwind utilities: standard Tailwind names
- Custom utilities defined globally in `globals.css`: kebab-case — `.chit`, `.dot-leader`, `.rule-hairline`, `.perforation`

**TypeScript path alias:**
- `@/*` resolves to `src/*` (configured in `tsconfig.json`)
- Use `@/components/...` and `@/app/...` for imports from within `src/`

## Where to Add New Code

**New page/route:**
- Create `src/app/<route>/page.tsx`
- Add `"use client"` if the page uses Convex hooks or `localStorage`
- Place any route-specific sub-components in `src/app/<route>/` or promote to `src/components/` if reused

**New shared component:**
- `src/components/<ComponentName>.tsx`
- Add `"use client"` at the top if the component uses hooks, browser APIs, or event handlers

**New Convex query:**
- Add to `convex/queries.ts` (create the file if it doesn't exist yet)
- Export as `export const myQuery = query({ ... })`
- Run `pnpm dev:convex` to regenerate `convex/_generated/`

**New Convex mutation:**
- Add to `convex/mutations.ts`
- Same export and regeneration pattern as queries

**New Convex action (side effects, external HTTP):**
- Add to `convex/actions.ts`

**New global CSS utility:**
- Add to `src/app/globals.css` under the appropriate section
- Use the existing token variables (`var(--color-ink)`, `var(--font-body)`, etc.)

**Static assets:**
- Place in `public/` — accessible at `/<filename>`
