<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→Next.js route | Dynamic billId in URL path — not validated at route level (Convex query returns null for invalid IDs) |
| localStorage→useQuery | organizerSecret read from localStorage and passed as query arg — null-guarded before passing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /dashboard/[billId] — organizerSecret from localStorage | mitigate | Query uses ternary skip (null secret = "skip") preventing request until secret loads; Convex server verifies secret in getBillForOrganizer |
| T-02-02 | Information Disclosure | /c/[billId] page — auto-redirect leaks that this device is the organizer | accept | Organizer owns the device; redirect is intentional UX per D-05 |
| T-02-03 | Tampering | billId URL parameter — user modifies path to access different bill | accept | getBillForMember is public by design (SHARE-04); getBillForOrganizer validates organizerSecret; no sensitive data exposed in public query |
| T-02-04 | Denial of Service | Invalid billId in URL | accept | useQuery returns null for non-existent Convex IDs; UI handles null state with "THIS CHIT HAS BEEN TORN UP" message |
| T-02-SC | Tampering | npm installs | accept | No new packages installed in this plan |
</threat_model>
