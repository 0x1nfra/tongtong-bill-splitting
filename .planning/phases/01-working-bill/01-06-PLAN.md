StatusBadge props:
  type StatusBadgeProps = Readonly<{

  }>

MemberRow props:
  type MemberRowProps = Readonly<{
    name: string
    status: "N/A" | "CONFIRMED" | "AWAITING" | "CLAIMED — UNPAID" | "UNCLAIMED ❋"
    amountOwed: number   // integer cents
    paymentId?: string   // present when payment exists — for confirm/reject
    onConfirm?: () => void
    onReject?: () => void
    onRemind?: () => void
  }>

ProgressBar props:
  type ProgressBarProps = Readonly<{
    collectedCents: number
    totalCents: number
  }>

StatsBar props:
  type StatsBarProps = Readonly<{
    confirmed: number
    awaiting: number
    claimed: number
    unclaimed: number
  }>

Dashboard status derivation (Phase 1 — no claiming):
  - For each payment in payments array:
    - status "settled" → member status = "CONFIRMED"
    - status "pending" → member status = "AWAITING"
    - status "rejected" → member status = "CLAIMED — UNPAID"
  - Members with no payment record: status = "UNCLAIMED ❋"
  - Organizer self: N/A (Phase 1 has no organizer-self detection; show all payment entries)
  Note: Phase 1 has no claim records to count, so CLAIMED count = payments with "rejected" status;
        UNCLAIMED count is not directly derivable without member count (show 0 or omit in Phase 1)

TOTAL COLLECTED calculation:
  collectedCents = sum of payment.amountOwed for all settled payments
  Phase 1: since no claiming, each settled member "owes" the full bill grand total ÷ number of members
  Simpler: collectedCents = count of settled payments × grandTotalCents (show as fraction of total if 1 member)
  For Phase 1 simplicity: TOTAL COLLECTED = number of confirmed payments × (grandTotalCents / total unique payment sessions)
  If no payments exist: collectedCents = 0

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→confirmPayment/rejectPayment | organizerSecret passed from localStorage; Convex verifies server-side |
| dashboard page | Page renders private bill data; guarded by organizerSecret which Convex verifies |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Spoofing | confirmPayment/rejectPayment called without valid secret | mitigate | Both mutations verify bill.organizerSecret !== arg.organizerSecret and throw "Unauthorized" (implemented in Plan 01 convex/payments.ts) |
| T-06-02 | Elevation of Privilege | /dashboard/[billId] page visible without correct secret | mitigate | getBillForOrganizer query throws "Unauthorized" if secret wrong; page shows "DASHBOARD NOT ACCESSIBLE" error; no data rendered |
| T-06-03 | Tampering | confirmPayment called on already-settled payment | accept | Convex DB patch is idempotent for same status; confirmedAt overwrites with newer timestamp — acceptable |
| T-06-04 | Denial of Service | Dashboard polls getPaymentsForBill continuously | accept | Convex useQuery is WebSocket subscription, not polling — free tier handles demo scale |
| T-06-05 | Information Disclosure | BillSummaryCard in right panel shows all items to organizer | accept | Organizer created the bill — they see all their own data |
| T-06-SC | Tampering | npm installs | accept | No new packages installed in this plan |
</threat_model>
