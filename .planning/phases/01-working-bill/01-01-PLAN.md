<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→Convex mutation | Client sends organizerSecret as plain argument; Convex verifies server-side |
| public query | getBillForMember returns bill data with no auth — intentionally public |
| member→payment | markPaid / getMyPayment keyed by claimantSession UUID — unguessable but not cryptographically authenticated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | confirmPayment mutation | mitigate | Server-side organizerSecret comparison: `bill.organizerSecret !== organizerSecret` throws before any write |
| T-01-02 | Spoofing | rejectPayment mutation | mitigate | Same organizerSecret check as confirmPayment |
| T-01-03 | Tampering | markPaid — duplicate payments | mitigate | Query by_session index first; return existing paymentId if pending/settled — no double-payment records |
| T-01-04 | Information Disclosure | getBillForMember returns organizerSecret | mitigate | Explicitly exclude organizerSecret from getBillForMember return value; only getBillForOrganizer returns it (after secret verify) |
| T-01-05 | Elevation of Privilege | getBillForOrganizer without secret | mitigate | Handler throws "Unauthorized" if organizerSecret argument does not match stored value |
| T-01-06 | Denial of Service | Convex free tier (1M calls/month) | accept | Demo scale; free tier sufficient per PROJECT.md constraint |
| T-01-SC | Tampering | npm installs | accept | No new packages installed in this plan; existing packages verified |
</threat_model>
