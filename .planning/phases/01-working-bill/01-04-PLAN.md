<!-- QR Upload 3-step pattern from RESEARCH.md Pattern 4 -->

Step 1: generateUploadUrl mutation → returns short-lived upload URL (string)
Step 2: fetch(uploadUrl, { method: "POST", headers: { "Content-Type": file.type }, body: file })
         → response.json() returns { storageId: string }
Step 3: pass storageId to createBill as qrStorageId argument

QRUpload props:
  type QRUploadProps = Readonly<{
    onUpload: (storageId: string) => void
  }>

BillSummaryCard props:
  type BillSummaryCardProps = Readonly<{
    title: string
    items: Array<{ name: string; price: number; quantity: number }>  // price in cents
    applySST: boolean
    applyServiceCharge: boolean
    displayCode: string  // e.g. "#TT-K97C"
  }>

CopyLinkField props:
  type CopyLinkFieldProps = Readonly<{
    url: string
  }>

Share screen URL to copy:
  `${window.location.origin}/c/${billId}`
  Note: window.location is browser-only — read inside onClick handler or useEffect, NOT at render time

WhatsApp share URL format:
  `https://wa.me/?text=${encodeURIComponent(shareUrl)}`
  (SHARE-04: recipients need no login; sharing the /c/[billId] URL is sufficient)

Calculating display code from billId:
  `#TT-${billId.slice(0, 4).toUpperCase()}`
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→Convex file storage | Client POSTs file directly to Convex storage URL generated by generateUploadUrl mutation |
| share URL→clipboard | navigator.clipboard.writeText is browser-only; called in onClick handler (safe) |
| WhatsApp link | URL contains /c/[billId] which is public by design (SHARE-04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | QRUpload — arbitrary file type upload | mitigate | input accept="image/*" limits file picker to images; Content-Type header set from file.type; Convex storage validates at server |
| T-04-02 | Information Disclosure | Share URL exposes billId | accept | billId is a Convex-generated unguessable ID; URL is intentionally shareable (SHARE-04) |
| T-04-03 | Spoofing | CopyLinkField — clipboard API unavailable (HTTP) | accept | navigator.clipboard requires HTTPS in production; dev on localhost works; Vercel deployment is HTTPS; no fallback needed for MVP |
| T-04-04 | Denial of Service | Large QR image upload (>10MB) | accept | Demo scale; Convex storage handles limits server-side |
| T-04-SC | Tampering | npm installs | accept | No new packages installed in this plan |
</threat_model>
