<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| member→markPaid | claimantSession is a localStorage UUID — unguessable but not cryptographically authenticated; anyone who knows the session can call markPaid |
| member view | getBillForMember is public — no auth; billId URL must be shared intentionally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | markPaid — member presents fake claimantSession | accept | Session is a UUID stored in localStorage; guessing another session's UUID is computationally infeasible; MVP trust model is sufficient per PROJECT.md |
| T-05-02 | Tampering | markPaid called multiple times (double-tap) | mitigate | Convex markPaid handler queries by_session index first — returns existing paymentId if pending/settled (idempotent per Plan 01 implementation) |
| T-05-03 | Information Disclosure | bill.qrUrl exposed to member view | accept | DuitNow QR is intentionally shared — it's how members pay. URL is a signed Convex storage URL. |
| T-05-04 | Denial of Service | Member submits empty claimantName | mitigate | Client-side guard: button disabled when claimantName.trim() === ""; handlePay returns early if empty |
| T-05-05 | Spoofing | Organizer opens /c/[billId] and sees member view instead of redirect | mitigate | Plan 02 organizer redirect useEffect: checks localStorage "tongtong_organizer_secret" → router.replace to dashboard |
| T-05-SC | Tampering | npm installs | accept | No new packages installed in this plan |
</threat_model>
