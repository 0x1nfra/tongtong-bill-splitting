# Phase 2 Plan 01: Convex Claim Mutations and Queries Summary

**One-liner:** claimItem/unclaimItem mutations with idempotency+session guards, getClaimsForBill query for dashboard CLAIMED/UNCLAIMED counts, and getBillForMember extended with claims[] for single-subscription member view.

## What Was Built

Added four Convex backend capabilities to `convex/bills.ts`:

1. **`claimItem` mutation** — inserts a record into the `claims` table. Idempotency guard: checks `by_session` composite index (billId + claimantSession) filtered by itemId before inserting; returns existing `_id` if found. Validates bill and item existence, and that item belongs to the bill.

2. **`unclaimItem` mutation** — deletes a claim record. Session-ownership gated: throws `"Unauthorized: can only unclaim your own items"` if `claim.claimantSession !== claimantSession`. Throws `"Claim not found"` if claim does not exist.

3. **`getClaimsForBill` query** — organizer-authenticated query returning `{ claimedCount, unclaimedCount }`. Auth failure returns `null` (WR-06 pattern). CLAIMED count = unique claimantSessions with ≥1 claim minus sessions with pending/settled payment. UNCLAIMED count = items with zero claim records.

4. **`getBillForMember` extended** — now queries `claims` table by `by_bill` index and includes `claims` in the return object. Single subscription covers all data the member view needs.
